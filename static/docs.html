API Reference Documentation

Welcome to the Leads Scraper v2 API reference. This service offers high-performance parallel Google Maps discovery and website enrichment using context page pooling.

Getting Started

All API requests should be sent to the base URL: http://localhost:8000. Response payloads are returned in JSON format.

Authentication

Security: Requests to all jobs and leads endpoints require the API Key to be passed inside the HTTP headers as X-API-Key.

The default secret key is change-me-in-production. Authentication is bypassed automatically for health checking and WebSocket connection handshakes.

Job Management

POST /jobs
Create and start a new background discovery and enrichment job immediately.

Request JSON Body

Parameter Type Status Description
name string required A descriptive title for the job.
keywords array[str] required Search terms (e.g. ["dentist"]).
areas array[str] required Search locations (e.g. ["London"]).
max_results integer optional (50) Limit of leads to return.
concurrency integer optional (3) Number of parallel workers.
curl -X POST http://localhost:8000/jobs \ -H "Content-Type: application/json" \ -H "X-API-Key: change-me-in-production" \ -d '{ "name": "Dentists in London", "keywords": ["dentist"], "areas": ["London"], "max_results": 10, "concurrency": 3 }'
GET /jobs
List paginated history of all scraping jobs. Can be filtered by job status.

Query Parameters

Parameter Type Status Description
status string optional Filter by status: pending, running, completed, failed, paused.
limit integer optional (50) Pagination page limit size.
offset integer optional (0) Pagination offset.
curl http://localhost:8000/jobs?limit=10 \ -H "X-API-Key: change-me-in-production"
GET /jobs/{job_id}
Fetch the configuration, status, and running details of a single job.

Path Variables

Variable Type Description
job_id UUID Unique identifier of the job.
curl http://localhost:8000/jobs/d09f6df8-1be2-44df-9d55-12cf5ee4e341 \ -H "X-API-Key: change-me-in-production"
DELETE /jobs/{job_id}
Deletes a job from the database. This cascades and automatically purges all collections (raw results) and enriched leads associated with the job ID. Cancels background execution first if the job is running.

Path Variables

Variable Type Description
job_id UUID Unique identifier of the job to delete.
curl -X DELETE http://localhost:8000/jobs/d09f6df8-1be2-44df-9d55-12cf5ee4e341 \ -H "X-API-Key: change-me-in-production"
POST /jobs/{job_id}/pause
Pause / Stop a running job by cancelling the background scheduler and scraping workers.

Path Variables

Variable Type Description
job_id UUID Job identifier to pause.
curl -X POST http://localhost:8000/jobs/d09f6df8-1be2-44df-9d55-12cf5ee4e341/pause \ -H "X-API-Key: change-me-in-production"
POST /jobs/{job_id}/resume
Resume a paused job, re-spawning parallel discovery and enrichment workers.

Path Variables

Variable Type Description

GET /jobs/{job_id}/queue-stats
Get the real-time active status, worker counts, and queue depth sizes of a pipeline job.

Path Variables

Variable Type Description
job_id UUID Identifier to check.
curl http://localhost:8000/jobs/d09f6df8-1be2-44df-9d55-12cf5ee4e341/queue-stats \ -H "X-API-Key: change-me-in-production"
WS /ws/jobs/{job_id}
Establish a persistent WebSocket connection to receive real-time JSON stream updates of job status, progress percentages, and queue depth modifications. Does not require X-API-Key header verification (supports browser WebSocket standard).

WebSocket Parameters

Path Variable Type Description
job_id UUID The unique job identifier to stream.
const socket = new WebSocket('ws://localhost:8000/ws/jobs/' + jobId); socket.onmessage = (event) => { const data = JSON.parse(event.data); console.log("Real-time Update:", data); };
